# Product

## Register
product

## Users
AI/ML 工程师、应用开发者、小团队技术成员。他们在配置、调试、调优一个 Agentic RAG 系统时使用 v-rag 管理端：编排意图路由图、上传与索引知识库文档、在 playground 调试对话并审查 agent 决策链、查看与纠错长期记忆、运行评测。这是团队内部共享工具:使用频繁、专业、需要快速到达任何功能，明暗环境都会用到。

## Product Purpose
v-rag 是开源 Agentic RAG 系统，管理端是它的配置与调试控制台。让用户无需改代码即可:可视化编排意图路由图、调试 agent 的每一步决策、管理多模态知识库、查看与编辑长期记忆、评测回答质量。设计服务产品:把 agent 内部复杂的意图路由、记忆召回、自主规划，变成可观察、可配置、可调试的对象。

## Brand Personality
克制、精确、可信 (restrained, precise, trustworthy)。Linear-like 的安静专业感:不喧哗、不炫技，把注意力让给内容 (对话、trace、路由图)。键盘优先，留白充足，信息有节奏。冷静，但不冷漠。

## Anti-references
- Dify / FastGPT / RAGFlow 那类 RAG 平台的模板感与套路布局
- 传统企业 B2B 后台的沉闷:密集表格、未设计的表单、千篇一律的侧栏加顶栏
- AI 紫色渐变、科技感发光、滥用玻璃态
- 信息过载与花哨动效 (为动而动)

## Design Principles
1. Content first, chrome second. 注意力让给对话、trace、路由图，UI 框架退后。
2. Calm by default. 安静、留白、有节奏，不靠动效与色彩制造存在感。
3. Observable, not opaque. agent 的每一步决策都可看见、可追溯 (节点级 trace)。
4. Keyboard as first-class. 高频操作键盘可达，命令面板优先。
5. Show the thinking. 把意图路由、记忆召回、自主规划这些黑盒变成可视、可调试的对象。

## Accessibility & Inclusion
- WCAG AA (团队内部工具，但开源，应达标)
- 明暗双主题都为一等公民 (团队不同环境)
- 支持 prefers-reduced-motion
- 严格对比度:trace、代码、数据要清晰可读
